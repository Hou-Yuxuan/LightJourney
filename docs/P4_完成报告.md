## 三、功能实现详情

### 3.1 行程列表页 (TripList.vue)

- **筛选**：支持按城市（文本输入）、日期范围（daterange 选择器）独立或组合筛选，条件变更即时刷新
- **日期分组**：按日期降序（新→旧）排列，每天显示星期几 + 当日预算小计
- **预算统计**：顶部筛选栏右侧显示总预算，每个日期标题右侧显示当日预算
- **空状态**：无行程时显示 `el-empty` + "让 AI 帮你规划"引导按钮
- **删除确认**：`ElMessageBox.confirm` 二次确认，不可恢复

### 3.2 行程卡片 (TripCard.vue)

- 左侧时间区 + 右侧内容区布局
- hover 显示编辑/生成文案/删除三个操作按钮（`opacity: 0 → 1` 动效）
- 通过 emit 向上传递 edit / delete / copywriting 事件

### 3.3 行程编辑弹窗 (TripFormDialog.vue)

- 新建/编辑复用同一组件，通过 `trip` prop 区分模式
- 前端校验：城市、日期、时间、标题必填，结束时间必须晚于开始时间
- 调用 createTrip / updateTrip API，409 冲突由拦截器统一提示

## 四、设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 日期排序 | 降序（新→旧） | 用户更关心即将出发的行程 |
| 编辑/新建复用 | 同一弹窗 + watch 切换 | 减少代码重复，维护方便 |
| 操作按钮 | hover 显示 | 极简设计原则，日常浏览不打扰 |
| 删除确认 | ElMessageBox.confirm | Element Plus 原生组件，风格统一 |

## 五、与 P5 协作

TripList.vue 集成了 P5 的两个 AI 弹窗：
- `AIPlanDialog`：点击"AI 帮我规划"按钮触发
- `AICopywritingDialog`：从 TripCard 的 `@copywriting` 事件触发，传入选中行程
