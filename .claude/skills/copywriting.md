# Skill2 — 行程文案生成

## 触发场景

用户选中一条行程想生成朋友圈分享文案时触发，例如：
- "帮我给这个行程生成一段朋友圈文案"
- "给大熊猫基地那条行程写个分享文案"
- "帮我把今天的行程写成朋友圈"

## API 端点

**生成文案：** `POST /api/ai/copywriting`
```
Authorization: Bearer <token>
Content-Type: application/json

{
  "trip_id": 1              // 必填，目标行程 ID
}
```

**响应：**
```json
{
  "code": 200,
  "data": {
    "copywriting": "今天在成都暴走两万步🐼 早起去熊猫基地...",
    "trip_title": "大熊猫基地",
    "trip_date": "2026-08-15"
  }
}
```

## Agent 工作流

### Step 1: 确定目标行程
- 如果用户明确指定了 trip_id，直接使用
- 如果用户描述行程内容（如"大熊猫那条"），先从 GET /api/trips 查询列表匹配
- 如果用户没指定，列出最近的行程让用户选择

### Step 2: 调用 AI 生成
- 调用 `POST /api/ai/copywriting`，传入 trip_id
- 后端自动获取当天同城所有行程作为上下文，生成更丰富的文案
- 展示生成的文案供用户预览

### Step 3: 编辑 + 复制
- 文案展示在可编辑的文本框中，用户可以手动调整
- 提供"一键复制"按钮，复制到剪贴板
- 支持重新生成（点"重新生成"再调一次 API）

## 文案风格特点

DeepSeek 被配置为旅行博主人设，生成的口语化文案具有以下特点：
- **口语化**：像在跟朋友聊天，自然不做作
- **有情绪**：表达真实感受——惊喜、感动、放松、惬意
- **生活感**：写体验和感受，不写景点介绍或攻略
- **4-6 句**：篇幅精炼，串联当天行程
- **Emoji 克制**：仅 1-2 个贴切的 emoji

## 关键约束

1. **文案仅返回文本** — 不写入数据库，不修改行程
2. **需要登录** — API 调用需要 JWT token
3. **行程归属校验** — 只能为自己的行程生成文案，操作他人行程返回 403
4. **行程必须存在** — trip_id 无效返回 404
5. **DeepSeek 超时 30 秒** — 超时提示重试

## 示例对话

**用户：** 帮我给"大熊猫基地"那条行程写个朋友圈文案
**Agent：** 好的，来给熊猫基地写段分享 🐼
→ 调用 API → 展示文案：
> 今天特意起了大早赶去熊猫基地，结果滚滚比我还懒 🐼 有一只趴在树杈上睡得四仰八叉...下次来成都，我还要早起来看它们打滚。
→ 用户可编辑或一键复制

## 错误处理

| 状态码 | 含义 | 处理 |
|--------|------|------|
| 404 | 行程不存在 | 提示用户检查行程 ID |
| 403 | 无权操作 | 提示该行程不属于当前用户 |
| 502 | AI 服务异常 | 提示稍后重试，检查 API Key 配置 |
| 401 | 未登录 | 引导用户登录 |
